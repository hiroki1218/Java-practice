# Git / GitHub について

## Git

Git とは commit 単位でバージョンを追跡するツール。
ターミナルにてコマンドを用いて操作する。
以下の 3 つの作業環境がある。

- **作業ディレクトリ** — ファイルを編集する場所
- **ステージ**      — 次の commit に含めるファイルを待機させる場所
- **コミット（履歴）**   — スナップショットとして保存された状態

## GitHub

GitHub はリモート上で Git の履歴を保管・共有するサービス。
GUI で操作するのが一般的。

## 初回コミットにて使用したコマンド一覧

| コマンド                          | 説明                                                            |
| --------------------------------- | --------------------------------------------------------------- |
| `git init`                        | ローカルに Git リポジトリを初期化                               |
| `git add Hello.java`              | 作業ディレクトリからステージングエリアへ上げる                  |
| `git commit -m "最初のコミット"`  | ステージの内容をスナップショットとして保存                      |
| `git branch -M main`              | ブランチ名を強制的に変更（master → main）                       |
| `git remote add origin <URL>`     | ローカルにリモートリポジトリの URL を "origin" という名前で登録 |
| `git remote set-url origin <URL>` | 登録済みのリモート URL を変更                                   |
| `git push -u origin main`         | ローカルの main ブランチの履歴を origin（GitHub）に送信         |

## オプション説明

| オプション | 意味                                                |
| ---------- | --------------------------------------------------- |
| `-m`       | message を指定                                      |
| `-M`       | ブランチ名を強制変更                                |
| `-u`       | 追跡関係を設定（2 回目以降は `git push` のみで OK） |

## 補足

- **origin** — ローカルに登録したリモートリポジトリのニックネーム
- **main** — ブランチ名（commit の履歴が並ぶ一本の線）

## つまずいたポイント

### HTTPS と SSH の認証エラー

`git remote add origin <URL>` にてリポジトリ先を登録する際に URL が HTTPS 形式となっていた。
ローカルの接続方式は SSH のため認証方式が一致せずエラーが発生した。
そのため `git remote set-url origin <SSH-URL>` にて登録済みの URL を SSH 形式に変更した。

### ブランチ名の不一致

GitHub 上ではデフォルトブランチ名が `main` だが、ローカルの `git init` では `master` がデフォルトとなっている。
そのため push する際にブランチ名が異なりエラーが発生した。
`git branch -M main` を使用してローカルのブランチ名を GitHub に合わせて変更した。
